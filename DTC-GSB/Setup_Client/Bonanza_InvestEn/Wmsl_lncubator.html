ย้อนกลับ

      การกำหนดค่า Wmsl.Incubator.Configuration.UI.exe

        1. ไปที่ C:\Bonanza\Operation\Buc


        2. คลิกขวา Run as administrator ที่ Wmsl.Incubator.Configuration.UI.exe ตามรูป


        3. แสดงหน้าจอสำหรับกำหนด Environment Bonanza Universal Configuration


        กำหนดค่าใน Tab Application

        a. Bonanza Path = ค่าเดียวกับ Local Path ใน ENVSetup.EXE
        b. Local Path = ค่าเดียวกับ Local Path ใน ENVSetup.EXE
        c. Server Path = ค่าเดียวกับ Server Path ใน ENVSetup.EXE
        d. Layout Path = ค่าเดียวกับ Local Path ใน ENVSetup.EXE
        e. สามารถเปิด ENVSetup.EXE เพื่อ Copy ค่ามาใส่ได้


        กำหนดค่าใน Tab Database

        a. Database Provider ให้เลือก SQL Server
        b. Server ให้ตรงกับ DB SERVER ใน ENVSetup.EXE
        c. Port : ระบุ Port ของ SQL server
        d. Database Name ให้ตรงกับ DB NAME ใน ENVSetup.EXE
        e. Default Schema = ค่าว่าง
        f. User = ค่าว่าง
        g. Password = ค่าว่าง
        h. ODBC ให้ระบุเหมือน Database Name หรือ ODBC Name ตามที่กำหนดไว้


        กำหนดค่าใน Tab Interface

        a. Sever Name : Interface Server Name เหมือนใน ENVSetup.EXE
        b. Port เหมือนใน ENVSetup.EXE ใช้ค่า default ของระบบ
        c. Username: ใช้ค่า default ของระบบ
        d. Password: ใช้ค่า default ของระบบ


        กำหนดค่าใน Tab Winserv

        a. Winserv IP ระบุ URL หรือ IP ของ Server ที่ติดตั้ง Invest-Service API (InterfaceNetCore)
        b. Check box Https ติ๊กเลือกเมื่อมีการตั้งค่า Invest-Service API เป็นแบบ Https
        c. Winserv Port ระบ Port ID ที่ใช้เชื่อมต่อ Invest-Service API
        d. Gateway IP:PORT: ระบุ URL หรือ IP ของ Server ที่ติดตั้ง Gateway Service และ Port ID ตามด้วย “/api” (กำหนดค่าเหมือน Gateway_prefix)


        4. ทดสอบ Heartbeat ทั้งส่วน Bonanza Gateway และ Bonanza Winserv จะต้องสามารถใช้งานได้ดังรูปตัวอย่าง


        5. กด Save เพื่อ บันทึกค่า BUC จะถูกบันทึกที่ C:\Bonanza\Operation บันทึกไฟล์ชื่อ buc.propeties
